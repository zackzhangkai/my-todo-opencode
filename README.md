# README: HTML Todo App / 待办清单应用

## 概述
- 这是一个纯前端的待办应用，使用 HTML/CSS/JavaScript 实现，特点是极简结构与高水平的美学设计，灵活可定制的主题风格。
- 仓库归属：zackzhangkai（GitHub 用户名）下的 my-todo-opencode 仓库（示例用途）。

## 快速开始
- 本地打开：直接在浏览器打开 todo.html 即可看到应用页面。
- 如果仓库中包含 README、LICENSE，请先浏览后再使用。

## 项目结构（当前实现）
- todo.html：主页面，包含 HTML、CSS（内联样式）与 JavaScript。
- 无其他外部依赖，无构建步骤即可直接在浏览器中使用。

## 设计方向与自定义
- 设计方向：极简中透出强烈美学，采用温暖复古调色板、纸张质感背景、渐层与阴影，强调排版与留白。
- 主题与变量：全局使用 CSS 变量，便于修改颜色、字体与间距。
- 色彩与字体：选择独特的字体与配色，避免常见的系统字体和乏味的配色。

## 功能特性
- 添加待办：输入文本后按回车或点击“添加”按钮将待办项添加到列表。
- 标记完成：点击圆点形复选框切换完成状态，完成项会显示删除线并改变样式。
- 编辑：双击文本进入编辑模式，按回车或离开焦点保存修改。
- 删除：点击删除按钮移除相应待办项。
- 筛选：全部、待办、已完成三种筛选模式。
- 清除已完成：一键删除所有已完成待办项。
- 交互：以微交互实现 hover 与 fokus 的视觉反馈。

## 可访问性与无障碍
- 使用清晰的对比度、可聚焦的控件、可操作的键盘导航。
- 图标与文本结合，方便屏幕阅读器解析。

## 如何自定义
- 修改 CSS 变量实现主题切换，例如主色、强调色、背景等。
- 通过修改字体或引入网格布局可进一步定制布局。

## 运行与部署（静态站点）
- 作为静态页面部署：将 todo.html 放在 Web 服务器的根目录即可访问。
- 如需集成到静态站点生成器，可将相关风格与脚本抽离为独立文件后再引入。
